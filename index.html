One-record cross-boundary failure demo

One filing record is tracked from source data to validation. Each local package checks only a node-specific condition. The comparison on the right shows how the same record can pass local checks and still become temporally illegal after a cross-boundary hand-off.

Shared record used in every step: period_end=2024-03-31, filed_at=2024-05-10, available_time=2024-05-10, decision_time=2024-04-01, label_window=[2024-04-02, 2024-04-08], eps_surprise=+0.18.
One record, one timeline, one violation: the record is only legally available on 2024-05-10, but downstream code tries to use it at decision_time=2024-04-01. Local packages can still pass because each one checks a narrower condition than the full cross-boundary contract.
